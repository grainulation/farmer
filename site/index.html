Get started See the dashboard

What is farmer?

When you run an AI coding agent like Claude Code, it sometimes needs your permission before executing commands or editing files. Normally you have to be sitting at your terminal. Farmer gives you a web dashboard that streams permission requests in real time over WebSocket. You can approve or deny from your phone, a different browser tab, or even while you are in a meeting. Zero external dependencies — standard library only.

Quick start — 3 steps
1

Start the server

Launch Farmer from your project directory. It prints a token-authenticated URL.

$ npx @grainulator/farmer start
Farmer running on port 9090 Dashboard: http://localhost:9090?token=a1b2c3 Open this URL on your phone or another tab.
2

Configure hooks

Point your Claude Code settings at Farmer's hook endpoint.

// .claude/settings.json
{
  "hooks": {
    "PreToolUse": [{
      "type": "command",
      "command": "curl -sf -X POST -H 'Content-Type: application/json' -d \"$(cat)\" http://localhost:9090/hooks/permission"
    }]
  }
}
Add this to .claude/settings.json. Hooks intercept tool calls before execution and route them to Farmer for approval.
3

Open the dashboard

Open the token URL in your browser or phone. Permission cards stream live as your agent works.

# Open the URL from step 1
# Cards stream in real time
# Approve or deny with one tap
Cards appear the instant your agent requests a tool call. No polling delay.

Key features

Control without context-switching

Zero external dependencies. Standard library only. No framework, no build step.

Token Authentication

Every session gets a unique token. No token, no access. The URL printed at startup is the only way in.

WebSocket Live Updates

Permission cards stream to your browser the instant they are requested. No polling, no lag. You see every tool call in real time as wheat sprints run.

Multi-Session

Run multiple AI agents at once. Each gets its own session in the sidebar. Switch with one click. When running parallel sprints, orchard coordinates permissions across all of them.

Trust Tiers

Paranoid, Standard, or Autonomous. Control how much your agent can do without asking. barn auto-detects which sprint your agent is working in so trust rules apply to the right context.

Audit Logging

Every approve, deny, and auto-decision is logged with timestamps. Full accountability.

Mobile Ready

Responsive layout works on your phone. Approve from the couch, the bus, or a meeting.

Frequently asked questions

Do I need Node.js?

Yes, Node 18 or later. But your project can use any language — Farmer works alongside any codebase.

Does this only work with wheat?

No. Farmer works with any Claude Code session. It hooks into the tool-call permission system, which applies to all agents, not just wheat sprints.

Does this work with agents besides Claude Code?

Currently farmer is built for Claude Code's hook protocol, but the architecture is agent-agnostic. Any tool that can POST a JSON permission request to an HTTP endpoint can use farmer.

How do I access farmer from my phone?

farmer binds to localhost by default. To access it from your phone or another device, expose it via a tunnel (cloudflared, ngrok, tailscale) or bind to 0.0.0.0 to serve on your local network. Push notifications work via the built-in PWA — add the dashboard to your home screen for native-like alerts.
